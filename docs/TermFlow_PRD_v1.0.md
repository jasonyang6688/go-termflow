## 一、产品概述

### 1.1 产品定位

TermFlow 是一款面向开发者、运维工程师及技术团队的新一代跨平台 SSH 终端工具，使用 Golang + Wails v2 构建。它不是对现有工具的简单复刻，而是从用户真实痛点出发，融合 FinalShell 的 SFTP 文件管理体验与 Electerm 的快捷命令标签机制，同时引入智能化连接保活、沉浸式终端体验等创新功能，打造一个「用起来特别爽」的现代化终端工具。

> **愿景**：让每一次连接服务器都像打开本地文件夹一样简单、流畅、不中断。

### 1.2 核心设计理念

| 理念 | 核心主张 | 体现方式 |
|------|---------|---------|
| 🧲 零摩擦 | 操作路径最短化，减少鼠标点击次数 | 快捷命令一键触发，SFTP 拖拽直传 |
| 🔒 永不断连 | 连接稳定是最基本的保障 | 智能心跳 + 自动重连 + 会话恢复 |
| ⚡ 高效率 | 常用操作触手可及，减少重复劳动 | 命令模板库、多标签批量操作 |
| 🎨 赏心悦目 | 工具也可以有品质感 | 主题系统、毛玻璃效果、流畅动画 |

### 1.3 竞品对比分析

| 功能 | FinalShell | Electerm | TermFlow ✨ |
|------|-----------|---------|------------|

| 快捷命令标签 | ❌ 无 | ✅ 有 | ✅ 增强版可分组 |
| 连接保活 | ✅ 有 | ⚠️ 易断连 | ✅ 智能自动重连 |
| 免费开源 | ⚠️ 部分收费 | ✅ 免费 | ✅ 完全免费 |
| 系统监控面板 | ✅ 图形化 | ❌ 无 | ✅ 实时图表 |
| 多标签管理 | ✅ 有 | ✅ 有 | ✅ 分组管理 |
| 命令历史搜索 | ⚠️ 基础 | ⚠️ 基础 | ✅ 智能模糊搜索 |
| 主题/皮肤 | ⚠️ 有限 | ✅ 多主题 | ✅ 完全自定义 |

## 二、目标用户与使用场景

### 2.1 目标用户画像

#### 👨‍💻 后端开发工程师
- 每天需登录多台开发/测试/生产服务器
- 频繁上传代码包、配置文件
- 有一套固定的部署/检查命令流程
- **痛点**：连接断了重来很烦，上传大文件经常失败

#### 🔧 运维 / DevOps 工程师
- 管理数十乃至上百台服务器
- 需要实时监控服务器资源状态
- 批量执行标准化运维命令
- **痛点**：工具分散，在多个窗口间来回切换效率低

#### 🎓 学生 / 独立开发者
- 自建服务器或使用云服务器
- 预算有限，希望用免费开源工具
- 希望工具好看、好用、上手快
- **痛点**：现有免费工具体验差，付费工具买不起

#### 👥 小型技术团队
- 需要共享服务器连接配置
- 希望统一团队的操作规范和命令集
- 需要导出/导入连接配置
- **痛点**：团队配置无法同步，每人维护一套

### 2.2 核心使用场景

#### 场景 A：大文件上传中途不断网
用户正在上传一个 2GB 的 Docker 镜像压缩包到服务器，上传进行到 70% 时网络抖动了一下。传统工具此时会失败并要求重新上传。TermFlow 通过断点续传机制，重新连接后自动从 70% 处继续，无需人工干预。

#### 场景 B：一键执行标准化部署流程
运维工程师为每个项目配置了一组快捷命令标签：「重启 Nginx」「查看日志」「清理缓存」「检查磁盘」。每次操作只需点击对应标签，告别手打命令，杜绝拼写错误带来的故障。

#### 场景 C：多服务器并行排查问题
故障发生时，工程师同时打开 5 个标签页连接不同服务器。TermFlow 的分组标签和同步发送功能，允许对选定的服务器组同时发送同一条命令（如 `tail -f /var/log/app.log`），在一个界面内对比多台服务器的输出。

#### 场景 D：低网络环境下的稳定操作
用户在地铁上用手机热点连接服务器，网络质量极差，频繁断线。TermFlow 的智能重连机制会在后台自动尝试重连，重连成功后恢复之前的会话状态，用户几乎感知不到中断。

### 3.2 SSH 连接管理

#### 3.2.1 连接配置
- 支持 SSH 密码认证、SSH Key（RSA/ED25519/ECDSA）、跳板机（ProxyJump）三种认证方式
- 连接支持按「项目」「环境」「标签」多维度分组，支持拖拽排序
- 连接列表支持搜索过滤，快速定位目标服务器
- 支持批量导入 `.ssh/config` 文件，兼容主流格式
- 支持导出为加密 JSON 包，用于备份或团队分享
- 连接可设置备注信息和自定义颜色图标，一眼区分环境（生产/测试/开发）

#### 3.2.2 智能保活与自动重连（核心创新）

> 💡 **设计亮点**：这是解决 Electerm 断连痛点的核心功能，用户永远不需要手动重连。

- 客户端每 60 秒自动发送 `keepalive@openssh.com` 心跳包
- 网络异常时自动触发重连，默认重试 10 次，间隔指数退避（1s → 2s → 4s…）
- 重连成功后自动恢复上次的工作目录、Shell 状态（通过会话记录机制）
- SFTP 传输任务在断连期间自动暂停，重连后继续（不丢失进度）
- 底部状态栏实时显示连接状态：🟢 连接中 → 🟡 重连中 → 🔴 失败
- 支持用户自定义重连次数和超时阈值

---

### 3.3 快捷命令标签系统（核心创新）

> 💡 **设计灵感**：来自 Electerm 的标签功能，TermFlow 将其升级为结构化的命令管理系统，支持分组、参数化和团队共享。

#### 3.3.1 命令标签基础功能
- 每个连接可配置专属命令标签栏，显示在终端下方
- 标签以「药丸形」按钮展示，支持自定义颜色和图标
- 点击标签立即将命令发送到终端并执行（无需回车确认模式可选）
- 支持「发送不执行」模式：命令填入输入框但不执行，方便修改参数后再回车
- 标签支持拖拽排序，使用频率高的放前面

#### 3.3.2 参数占位符（创新功能）
- 命令模板支持 `{{变量名}}` 语法，点击时弹出轻量输入框填写参数
- 示例：`tail -n {{行数}} {{日志路径}}`，点击后快速填参即可执行
- 参数支持设置默认值，多次使用会记住上次填写的值

#### 3.3.3 命令库管理
- **全局命令库**：跨所有连接共享的通用命令集（如系统检查、日志查看）
- **项目命令库**：某个项目组专用的命令集
- **连接命令库**：仅该连接可见的私有命令
- 支持以 JSON 格式导出/导入整个命令库，方便团队分享
- 内置常用命令模板：系统信息、进程查看、磁盘清理、Docker 管理等

---

### 3.4 SFTP 文件管理（核心创新）

#### 3.4.1 双面板文件管理
- 左侧本地文件系统，右侧服务器文件系统，双面板 Commander 风格布局
- 支持直接拖拽文件从本地面板到服务器面板，反之亦然
- 文件列表显示：名称、大小、修改时间、权限，支持排序和筛选
- 支持在服务器端直接预览文本文件、图片文件（无需下载）
- 右键菜单：新建文件/文件夹、重命名、压缩、解压、修改权限

#### 3.4.2 智能断点续传（核心亮点）

> 💡 **技术实现**：通过记录已传输的字节偏移量（offset），断线重连后从该偏移量处继续写入，完全透明无感。

- 每个传输任务记录已完成的字节偏移量，持久化到本地数据库
- 网络中断时任务自动暂停并标记进度，重连后无需用户操作自动续传
- 支持用户手动暂停/继续单个传输任务
- 传输队列支持优先级调整，重要文件插队
- 传输完成后自动校验 MD5/SHA256，确保文件完整性

#### 3.4.3 批量传输体验
- 多选文件一键上传/下载，显示总进度条和单文件进度条
- 传输速度实时显示（KB/s 或 MB/s），预估剩余时间
- 传输完成后系统通知提醒（可配置是否开启）
- 传输历史记录，可查看过去 30 天的传输日志

### 4.2 核心交互流程

#### 4.2.1 新建连接（30 秒完成）
1. 点击左侧「+」按钮或按 `Cmd+N` / `Ctrl+N`
2. 在弹出的快速创建面板中填写：Host、Port、User（支持从剪贴板自动解析 SSH 命令）
3. 选择认证方式，密钥模式支持从 `~/.ssh/` 自动扫描可用密钥
4. 点击「测试连接」，3 秒内反馈结果
5. 连接成功后自动打开新终端标签

#### 4.2.2 快捷命令配置
1. 在标签栏右侧点击「⚙ 管理命令」
2. 点击「+ 新增命令」，输入名称、命令内容、选择颜色
3. 对于带参数的命令，使用 `{{参数名=默认值}}` 语法
4. 拖拽调整排序，点击「保存」
5. 命令标签立即出现在终端下方，点击即可执行

#### 4.2.3 SFTP 文件上传
1. 在终端标签的工具栏点击「📁 SFTP」图标，右侧展开文件面板（不遮挡终端）
2. 左侧本地文件夹导航到目标文件，右侧已自动定位到服务器当前路径
3. 拖拽文件到右侧面板，或点击「上传」按钮，开始传输
4. 底部传输队列实时显示进度，可继续在终端操作，不影响传输
5. 传输完成弹出轻提示（不打断工作流）

### 5.4 核心代码示例

**SSH 智能保活：**

```go
func (m *Manager) keepAlive(client *ssh.Client, connID string) {
    ticker := time.NewTicker(60 * time.Second)
    defer ticker.Stop()
    for range ticker.C {
        _, _, err := client.SendRequest("keepalive@openssh.com", true, nil)
        if err != nil {
            m.triggerReconnect(connID) // 触发自动重连
            return
        }
    }
}
```

**SFTP 断点续传：**

```go
func (s *Engine) resumeUpload(local, remote string, offset int64) error {
    remoteFile, _ := s.client.OpenFile(remote, os.O_WRONLY|os.O_CREATE)
    defer remoteFile.Close()
    remoteFile.Seek(offset, io.SeekStart)

    localFile, _ := os.Open(local)
    defer localFile.Close()
    localFile.Seek(offset, io.SeekStart)

    _, err := io.Copy(remoteFile, localFile)
    return err
}
```

## 七、非功能性需求

### 7.1 性能指标

| 指标 | 目标值 | 备注 |
|------|-------|------|
| 应用启动时间 | < 1.5 秒 | 冷启动到可交互状态 |
| SSH 连接建立 | < 2 秒 | 密码认证，正常网络 |
| 终端输入延迟 | < 16ms | 本地 → 远程 → 回显 |
| SFTP 上传速度 | ≥ 90% 带宽利用率 | 不因工具引入额外损耗 |
| 内存占用（空闲） | < 80MB | 无活跃连接时 |
| 内存占用（10连接） | < 300MB | 10 个活跃 SSH 连接 |
| CPU 占用（空闲） | < 1% | 后台无任务时 |
| 安装包大小 | < 50MB | 含所有依赖 |

### 7.2 安全性要求

- 所有敏感数据（密码、私钥）使用 **AES-256-GCM** 加密后存储于本地 SQLite，密钥由用户主密码派生（PBKDF2）
- 应用内存中的密钥在连接关闭后立即清零，不驻留
- **不上传任何数据到外部服务器**（完全离线工具，无遥测）
- 支持为应用设置访问密码锁，防止他人打开
- SSH `known_hosts` 校验强制开启，防止中间人攻击（可由用户手动信任）
- 私钥文件权限自动检查（chmod 600），权限不正确时给出警告

### 7.3 可靠性要求

- 连接配置本地自动备份，每次修改保留最近 10 个历史版本
- 应用崩溃后自动重启，崩溃日志写入本地文件供排查
- SFTP 传输任务状态持久化，即使应用崩溃重启后仍可继续未完成的传输
- 所有配置文件定期自动备份到用户指定目录

### 7.4 兼容性要求

| 平台 | 最低版本 |
|------|---------|
| macOS | 12 Monterey |
| Windows | Windows 10 21H2 |
| Linux | Ubuntu 20.04 / Debian 11 |
| SSH 服务端 | OpenSSH 7.0+ |
